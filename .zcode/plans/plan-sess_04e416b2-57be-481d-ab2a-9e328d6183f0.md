# Settings UI/UX Overhaul: Instant Open + Modern Look + Profile Fixes

## Root-cause findings (verified)

**1. Open delay** — `SettingsWorkspace.__init__` synchronously builds the embedded `SettingsCenterDialog`, which constructs **all 9 tabs** eagerly (dialogs.py:320-333). The `media_tools` tab alone takes ~96ms because `_refresh_media_dependency_status()` probes `yt_dlp`/`yt_dlp_ejs` via `importlib.import_module` and `ffmpeg`/`deno` via `shutil.which` on the main thread (dialogs.py:1279, 3788-3814). The whole build blocks the UI thread, so Settings feels frozen then pops in.

**2. "Token activity shows no data"** — `TokenHeatmapWidget.paintEvent` (settings_workspace.py:185-186): the grid is 53 weeks × 11px + 52 × 4px gap = **791px** plus a 42px left margin = 833px minimum width. But `sizeHint` returns only 760px (line 117) and there is **no `setMinimumWidth`** (line 113). The recent-activity days (late June / July) land in columns 49-52, painted at x=777-833 — **beyond the widget's visible right edge**. They're rendered off-widget and invisible. The user sees an empty grid. *This is the actual bug, confirmed numerically.*

**3. "Streaks/dates wrong"** — `get_profile_usage_snapshot()` (main_window.py:7939-7940) is **not read-only**. Its first line calls `_record_profile_session_snapshot()` → `_update_profile_streak()`, which advances `last_active_date` to today and **increments `current_streak` on the first Settings-open of each day — with zero actual usage**. Viewing the profile permanently mutates persisted state.

**4. "Stats don't update"** — The stat-label wiring is actually correct (verified). Stats appear stale only because the heatmap clipping makes the page look empty. Fixing #2 + #3 resolves this.

## Implementation plan

### A. Instant open (settings_workspace.py + dialogs.py)

1. **Defer embedded `SettingsCenterDialog` construction** in `_build_settings_page()` (settings_workspace.py:541-581). Instead of building the dialog synchronously, create the host with a lightweight "loading" placeholder, and build the real dialog via `QTimer.singleShot(0, self._materialize_settings_dialog)` *after* the workspace is shown. This lets the profile/help pages (and the workspace chrome) paint instantly; the heavy dialog builds on idle cycles without blocking the open click.

2. **Make `media_tools` probe lazy** (dialogs.py:1279): replace the synchronous `self._refresh_media_dependency_status()` call in `_create_media_tools_tab` with a `QTimer.singleShot(0, self._refresh_media_dependency_status)` so `yt_dlp`/`ffmpeg` detection doesn't block dialog open. This is independently a 96ms win even without the rest.

3. **Build tabs progressively**: optionally split the 9 `_create_*_tab` calls so they're added to a job queue processed one-per-event-loop-tick via `QTimer`. Keeps `_commit_settings` safe (all tabs exist by the time the user could click Save) while never blocking the open interaction. `_apply_settings_styles()` is already lazy-safe (uses `findChildren`), so late-built tabs still get styled.

### B. Profile / heatmap fixes (settings_workspace.py + main_window.py)

4. **Fix heatmap clipping** (settings_workspace.py:113-117): add `setMinimumWidth` derived from grid geometry (~grid_width + left margin + right margin, ~860px) so the enclosing `QScrollArea` engages horizontal scrolling instead of silently clipping. Also update `sizeHint` width to match. This makes the recent-activity columns visible.

5. **Make `get_profile_usage_snapshot()` read-only** (main_window.py:7939-7941): remove the `self._record_profile_session_snapshot()` call from inside the getter. Move session-snapshot recording to genuine lifecycle points (app startup, periodic save, app close) where it belongs — not into a view path. This stops the streak from inflating on Settings-open and stops persisted state from mutating on read.

### C. Modern look & feel (settings_workspace.py QSS + minor layout)

6. **Soften the visual style** in `_settings_workspace_qss()` (settings_workspace.py:858-1014):
   - Add subtle card backgrounds with slightly larger radii and faint borders for stat/insight cards (they currently render borderless against the bg — the "stiff/kaku" feeling).
   - Give the `sw-stat-strip` and `sw-info-card` gentle elevation (background = `panel`, border = `border`, radius = `lg`) so sections feel grouped rather than flat.
   - Improve the sidebar nav hover/selected states (add left accent bar + accent-tinted selected background) for clearer hierarchy.
   - Increase spacing in the profile page top identity block (currently `identity_l.setContentsMargins(0, 48, 0, 24)` is cramped against the avatar).

7. **Profile layout polish**: wrap the stat strip + insight/model cards in proper card frames (they use `sw-stat-strip` objectName but the insight cards use `sw-info-card` which is currently `background: transparent; border: none` — making them disappear). Give them real card styling.

### D. Verification & docs

8. **Verify**: `venv\Scripts\python.exe -m py_compile` on all changed files; offscreen smoke test (`QT_QPA_PLATFORM=offscreen`) that opens Settings, switches to profile, renders the heatmap with real `.cache/manga_ocr_usage_v16.json` data, and confirms the recent-activity squares are now visible (within widget bounds). Confirm streak does not change from a read-only open.

## Scope & risk
- All edits stay in `settings_workspace.py` (primary), plus two surgical edits in `dialogs.py` (media probe deferral) and `main_window.py` (make snapshot read-only). `main_window.py` changes are minimal per AGENTS.md §1.8.
- No new dependencies. No changes to settings.json schema. No changes to existing feature behavior.
- Existing tab content, save/commit logic, and data persistence all preserved — only *when* tabs build, *how* the heatmap is sized, and *that* a getter shouldn't write are changed.
- If `QTimer.singleShot` deferral of the dialog build proves risky (e.g., a caller expects the dialog to exist immediately after `_build_settings_page`), I'll fall back to building eagerly but moving only the media probe + profile refresh off the critical path — still a big perceived-speedup, lower risk.
